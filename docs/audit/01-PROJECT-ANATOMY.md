# Раскадровка — Анатомия проекта

> Полный аудит: из чего состоит проект, какие блоки, что работает, что нет.
> Дата: 5 апреля 2026

---

## Что это за продукт

**Раскадровка** (raskadrawka.ru) — веб-платформа для AI-продакшена. Пользователь создаёт проекты, генерирует изображения и видео через AI-провайдеров (сейчас Kie.ai), организует результаты в группы, делится с заказчиками для ревью.

**Целевая аудитория:** продюсеры, режиссёры, контент-мейкеры, рекламные агентства — те, кому нужно быстро визуализировать идеи через AI и согласовать их с командой.

**Ценностное предложение:** не просто "ещё один фронтенд к Midjourney", а **рабочее пространство** — проект → группы → элементы → шеринг → ревью. Полный цикл от идеи до согласованного результата.

### 1. Аутентификация и пользователи ****

**Что есть:**
- Регистрация с подтверждением email (Yandex SMTP)
- Вход по логину/паролю (JWT: 30 мин access, 7 дней refresh)
- Сброс пароля через email (токен на 1 час)
- Баннер "Подтвердите email" для неподтверждённых
- Страницы условий использования и политики конфиденциальности
- Квоты: лимиты на проекты (5), сцены (20), элементы (10), хранилище (300 МБ)

**Что хорошо:**
- Email enumeration protection (forgot-password не раскрывает, есть ли пользователь)
- Token rotation при refresh
- Rate limiting на auth-endpoints (5/мин)
- Dual storage (cookies + localStorage) для надёжности

**Что не хватает:**
- Двухфакторная аутентификация (2FA)
- OAuth (вход через Яндекс/VK/Google)
- Управление сессиями (список активных, выход из всех)
- CAPTCHA на публичных формах
- Индикатор сложности пароля при регистрации

### 2. Проекты ****

**Что есть:**
- CRUD проектов с названием и соотношением сторон (16:9 / 9:16)
- Статусы: Активный / На паузе / Завершён
- Карточки проектов с превью (1-4 thumbnail, компоновка адаптируется)
- Статистика: количество сцен, элементов, потраченных кадров, объём хранилища
- Grid с auto-fill, адаптивный к ширине экрана
- Skeleton-загрузка

**Что хорошо:**
- Карточки проектов красиво показывают содержимое (hero + side column)
- Статистика рассчитывается на бэкенде через аннотации (не N+1)

**Что не хватает:**
- Архивация вместо удаления (сейчас удаление безвозвратное)
- Дублирование проекта
- Шаблоны проектов
- Поиск по проектам
- Сортировка (по дате, по имени, по активности)

### 3. Группы (сцены) ****

**Что есть:**
- Иерархия: проект → группы (до 2 уровней вложенности)
- CRUD с drag-and-drop сортировкой
- Headliner — выбор элемента как обложки группы
- Статусы: Черновик / В работе / На ревью / Одобрено
- Breadcrumbs навигация

**Что хорошо:**
- Вложенность с валидацией глубины
- DnD через @dnd-kit работает плавно
- order_index синхронизируется с бэкендом

**Что не хватает:**
- Визуальное отделение групп от элементов в grid (запланировано, spec готов)
- Массовые операции с группами
- Workflow-автоматизация при смене статусов

### 4. Элементы и рабочая область *****

**Что есть:**
- Grid элементов с настройкой: размер карточки (compact/medium/large), соотношение сторон, режим заполнения
- Мультивыбор: Shift+click (диапазон), Ctrl+click (toggle), Select All
- Drag-and-drop сортировка внутри группы
- Массовые действия: удаление, перемещение в группу, избранное
- Фильтрация: все / избранные / изображения / видео
- Optimistic UI: элемент появляется в grid мгновенно при генерации
- Статусы элементов: pending → processing → completed / failed / uploading
- Overlay прогресса загрузки (фазы: resize → upload → completing)
- Skeleton-загрузка
- Empty state с CTA и поддержкой drag-and-drop

**Что хорошо:**
- Workspace store (~700 строк) управляет всем жизненным циклом элементов
- Upload queue с ограничением в 10 параллельных загрузок
- WebSocket обновляет карточки в реальном времени
- Настройки отображения сохраняются в localStorage

**Что не хватает:**
- Отмена генерации (нет cancel)
- Undo для удалённых элементов
- Batch download (ZIP-архив)
- Поиск по промптам
- Теги / метки для элементов

### 5. AI-генерация *****

**Что есть:**
- Выбор модели из каталога (IMAGE / VIDEO), фильтрация по типу
- PromptBar: auto-resize textarea, до 6 строк
- Image inputs: динамическая форма из `image_inputs_schema` (два формата: simple array, groups с зависимостями)
- Параметры генерации из `parameters_schema` (aspect ratio, quality, duration и др.)
- Оценка стоимости в реальном времени (POST /api/credits/estimate/)
- Проверка "хватает ли баланса" перед генерацией
- Retry из failed-элемента (восстанавливает prompt + параметры)
- ConfigPanel: collapsible sidebar с параметрами

**Что хорошо:**
- Schema-driven UI: ни одного хардкода параметров, всё из бэкенда
- Canonical Parameter System: переиспользуемые определения (resolution, aspect_ratio), привязки к моделям
- Request schema templating: `{{prompt}}`, `{{width}}` — подставляются при генерации
- Response mapping: настраиваемые JSON-пути для разных провайдеров
- Dual-mode polling: webhook + polling как страховка
- Атомарное списание кредитов при старте, идемпотентный рефунд при ошибке

**Что не хватает:**
- Negative prompt (как collapsible секция)
- Prompt suggestions / чипы-подсказки
- "Улучшить промпт" через AI
- Preset стили (визуальные карточки для быстрого выбора)
- Сравнение результатов (A/B view)
- Variations (генерация вариаций от понравившегося)
- Очередь генераций с приоритетами

### 8. Кредиты и финансы *****

**Что есть:**
- Баланс пользователя (DecimalField для точности)

- Идемпотентный рефунд при ошибке (не дублируется)

- Pricing: fixed или lookup (по комбинации параметров)
- pricing_percent: наценка/скидка на уровне пользователя

- Отображение баланса в header, в ConfigPanel, в PromptBar

**Что хорошо:**

- Гибкая система ценообразования (fixed + lookup + per-user markup)

**Что не хватает:**
- Пополнение баланса (нет интеграции с платёжкой)
- История списаний в удобном виде (в кабинете есть, но без группировки)
- Уведомление "баланс заканчивается"
- Промокоды / бонусы

### 10. Загрузка файлов ****

**Что есть:**
- Двухфазная загрузка: presigned URLs → клиент грузит напрямую в S3
- Клиент генерирует 3 версии: original + small (256px) + medium (800px)
- Upload queue: max 10 параллельных, AbortController для отмены
- Progress overlay с фазами (resize → presign → upload_thumb → upload_full → completing)
- Drop zone в empty state

**Что хорошо:**
- Сервер не пропускает через себя файлы — клиент грузит прямо в S3
- Thumbnails генерируются на клиенте (экономит серверные ресурсы)
- Очередь с ограничением параллельности

**Что не хватает:**
- Drag-and-drop на весь workspace (не только empty state)
- Pause / resume для загрузок
- Валидация файлов до загрузки (формат, размер)
- Глобальный индикатор загрузок (floating pill)
- Видео: извлечение metadata (длительность, разрешение)

## Архитектурные решения — что реально круто

### 1. Schema-driven AI UI
Ни одного хардкода параметров моделей. Всё из бэкенда: `parameters_schema`, `image_inputs_schema`, `pricing_schema`. Добавление новой модели — только через Django Admin, фронтенд подхватывает автоматически. Это **ключевое конкурентное преимущество** для масштабирования.

### 2. Canonical Parameter System
Параметры не привязаны к конкретной модели. `CanonicalParameter` (resolution, aspect_ratio) переиспользуется через `ModelParameterBinding`. Это позволяет единообразно работать с параметрами разных провайдеров.

### 3. Response Mapping
Каждый AI-провайдер возвращает данные в своём формате. `response_mapping` с JSON-путями нормализует ответы без изменения кода. Подключение нового провайдера — конфигурация, не разработка.

### 4. Кредитная система
Атомарный дебет → генерация → рефунд при ошибке. Идемпотентность. Audit trail. Pricing compiler. Это enterprise-уровень финансовой логики.

### 5. Двухфазная загрузка
Presigned URLs позволяют клиенту грузить прямо в S3, минуя сервер. Staging + Celery для обработки. Сервер никогда не блокируется на I/O.

### 6. Optimistic UI
Элемент появляется в grid мгновенно при генерации. WebSocket обновляет карточку когда результат готов. Пользователь не ждёт — работает дальше.

## Архитектурные проблемы — что нужно исправить

### Критические

| Проблема | Описание | Влияние |
|----------|----------|---------|
| **Секреты в git** | `.env` с production-данными в репозитории | Безопасность: все ключи скомпрометированы |
| **Нет remote backup** | Бэкапы только на VPS, нет копии вовне | Потеря данных при сбое сервера |
| **Нет платёжной интеграции** | Кредиты пополняются вручную | Невозможно монетизировать |
| **API-ключи в plaintext** | AIProvider.api_key хранится открыто в БД | Утечка при любом доступе к БД |

### Высокий приоритет

| Проблема | Описание |
|----------|----------|
| **Нет мониторинга** | Sentry ловит ошибки, но нет метрик, APM, alerting |
| **Email через threads** | Отправка email в отдельном потоке, а не через Celery — silent failures |
| **Нет CI/CD** | Деплой вручную через bash-скрипт |
| **Redis без persistence** | При перезапуске теряются все задачи в очереди |
| **Python version mismatch** | Dev: 3.12, Prod: 3.11 |
| **Нет API-документации** | Нет OpenAPI / Swagger |
| **Нет тестов** | Минимальное покрытие тестами |

### Средний приоритет

| Проблема | Описание |
|----------|----------|
| **Workspace store слишком большой** | 700+ строк, стоит разбить |
| **Generation state не сохраняется** | При обновлении страницы теряется модель, параметры, prompt |
| **Нет клиентской валидации** | Вся валидация на бэкенде |
| **Мобильная адаптация** | Sidebar не сворачивается, lightbox filmstrip обрезается |
| **Нет request deduplication** | Одинаковые GET-запросы дублируются |

## Резюме

**Раскадровка — это технически зрелый MVP.** Ядро продукта (генерация, организация, шеринг) работает хорошо и архитектурно правильно. Schema-driven подход к AI-моделям — это серьёзное конкурентное преимущество, которое позволяет масштабироваться без переписывания фронтенда.

**Сильные стороны:**
- Архитектура backend: чистое разделение, TIER-система, services.py как единственный интерфейс
- Кредитная система: enterprise-уровень
- AI-интеграция: гибкая, расширяемая
- Frontend UI: профессиональный, консистентный дизайн-система
- Optimistic UI + WebSocket: мгновенный отклик

**Слабые стороны:**
- Операционная зрелость: нет мониторинга, CI/CD, remote backups
- Монетизация: нет платёжки, пополнение вручную
- Безопасность: секреты в git, API-ключи в plaintext
- Полнота: кабинет не отполирован, скачивание примитивное, нет onboarding

**Общая оценка: 7.5/10 для MVP, 5/10 для коммерческого запуска.**

Подробности о том, что нужно доработать для запуска — в документе `03-PRODUCTION-READINESS.md`.
Бизнес-модель и тарифы — в документе `02-BUSINESS-MODEL.md`.
