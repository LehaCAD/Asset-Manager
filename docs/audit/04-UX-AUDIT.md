# Раскадровка — UX-аудит и роадмап доработок

> Полный аудит пользовательского опыта. Что работает, что сломано, что нужно для запуска.
> Дата: 5 апреля 2026

---

## Общая оценка UX

**Раскадровка — визуально зрелый продукт.** Дизайн-система консистентна (oklch цвета, dark/light, shadcn/ui, Lucide иконки). Основные workflow реализованы и работают. Но есть ряд UX-пробелов, которые будут болезненны для платящего пользователя.

**UX-зрелость по блокам:**

| Блок | Зрелость | Главная проблема |
|------|---------|-----------------|
| Генерация AI | 85% | Нет cancel, нет подсказок для новичков |
| Element Grid | 90% | Визуальное разделение групп и элементов |
| Lightbox | 80% | Нет zoom, нет pin-комментов |
| Шеринг | 75% | Пароль не показывается, нет уведомлений |
| Кабинет | 40% | Старый дизайн, нет скелетонов, ошибки молчат |
| Auth | 80% | Нет real-time валидации, нет OAuth |
| Onboarding | 0% | Не существует |
| Мобильная версия | 20% | Sidebar не сворачивается, filmstrip обрезается |
| Error handling | 30% | Много silent failures |

## Критические UX-проблемы

### 1. Нет onboarding — пользователь потерян

**Сейчас:** Регистрация → пустой экран "Нет проектов" → ???

**Нужно:**
- Бесплатные кадры при регистрации (5–10 штук)
- Направленные empty states: "Создайте первый проект" с одной CTA
- Чеклист: "Создать проект" → "Сгенерировать изображение" → "Поделиться"
- Prompt suggestions (3–4 чипа под пустым полем) для первой генерации

**Почему критично:** 8 из 10 пользователей бросают продукт, если не понимают, как им пользоваться. Без onboarding конверсия freemium → paid будет ниже 1%.

---

### 2. Silent failures — пользователь не знает, что пошло не так

**Где молчит:**
- Загрузка списка проектов — нет toast при ошибке
- Аналитика в кабинете — ошибка API не показывается
- Переключение избранного — нет loading indicator
- Set headliner — нет feedback
- Баланс кабинета — ошибка load не отображается

**Нужно:** Аудит всех API-вызовов → toast для ошибок → retry кнопка для failed загрузок.

---

### 3. Кабинет выглядит незаконченным

**Сейчас:**
- Не использует новый дизайн (v2026-04-04 утверждён, не реализован)
- Hardcoded цвета вместо токенов (красный/жёлтый для storage — нарушает правило "no fear colors")
- Нет loading skeletons для таблиц и графиков
- Нет error states
- Настройки минимальные (только пароль и тема)

**Нужно:** Полная реализация утверждённого дизайна. Это первое, что увидит платящий пользователь, когда захочет проверить баланс.

---

### 4. Workspace layout bug

**Проблема:** `h-screen` на main исключает высоту navbar. Контент может обрезаться или появляться лишний скролл.

**Решение:** `flex-1 min-h-0` или `h-[calc(100vh-48px)]`

## UX-проблемы по флоу

### Аутентификация

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Нет real-time валидации полей | Пользователь узнаёт об ошибке только после submit | Inline validation по blur |
| Нет проверки username availability | Конфликт при регистрации | Debounced check |
| Password strength indicator отсутствует | Слабые пароли | Визуальный индикатор |
| Forgot password скрывает все ошибки | Пользователь не знает, что пошло не так (даже network error) | Показывать хотя бы "Произошла ошибка, попробуйте позже" |
| Нет OAuth (VK, Яндекс) | Барьер входа для ленивых пользователей | Добавить VK ID + Яндекс ID |

### Проекты

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Нет error state при загрузке | Белый экран при ошибке | Toast + retry |
| Нет поиска по проектам | Теряется при 10+ проектах | Search bar |
| Нет сортировки | Нельзя найти "последний" | По дате, по имени, по активности |
| Удаление безвозвратное | Страх потери данных | Архивация (30 дней) |
| Share button грузит элементы синхронно | Медленно на больших проектах | Loading indicator / async |

### Рабочая область (Workspace)

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Группы и элементы в одном grid | Визуальная каша | Разделить: горизонтальная полоса групп + grid элементов (spec готов) |
| Нет cancel для генерации | Пользователь не может остановить случайную генерацию | Cancel API + UI кнопка |
| Upload — нет pause/resume | Большие файлы зависают | AbortController + resume |
| Drag-and-drop только в empty state | Хочу бросить файл на любой участок workspace | Глобальный dropzone |
| Нет keyboard shortcuts | Power users недовольны | Cmd+K, Cmd+A, Del, Enter для lightbox |
| Generation state теряется при refresh | Потеря модели, промпта, параметров | Persist в sessionStorage |

### Генерация

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Пустое поле промпта пугает новичков | "Что написать?" | Prompt suggestions — 3–4 кликабельных чипа |
| Стоимость не рядом с кнопкой генерации | Пользователь не видит, сколько спишется | Показать "2 Кадра" рядом с кнопкой |
| Нет negative prompt | Нет контроля "чего не хочу" | Collapsible секция в Advanced |
| Нет preset стилей | Каждый раз настраивать с нуля | Визуальные карточки стилей |
| Нет сравнения результатов | Сложно выбрать лучший | Side-by-side view в lightbox |

### Lightbox

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Нет zoom | Не видно деталей изображения | Wheel zoom + pan |
| Нет pin-комментариев | Нельзя указать на конкретное место | Клик → комментарий с координатами |
| Filmstrip обрезается на мобильном | Навигация сломана | Swipe-gesture, bottom sheet |
| Нет fullscreen | Отвлекающие элементы вокруг | Нативный Fullscreen API |
| Нет "следующий неотсмотренный" | Для workflow ревью приходится искать вручную | Кнопка "Далее (неотсмотренный)" |

### Шеринг

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Пароль: backend есть, UI нет | Функция невидима | Показать поле пароля в CreateLinkDialog |
| Нет уведомлений о комментариях | Автор не узнает о фидбеке | WS event уже есть → добавить UI |
| Comment resolved/unresolved нет | Нельзя отследить прогресс | Чекбокс "Решено" на комментарии |
| Нет email-приглашений | Нужно копировать ссылку вручную | "Пригласить по email" |
| Комменты не paginated | При 100+ комментариев — тормоза | Пагинация / infinite scroll |

### Кабинет

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Весь дизайн устарел | Не соответствует brandbook | Реализовать дизайн v2026-04-04 |
| Нет top-up UI | Нельзя пополнить баланс | Страница оплаты |
| Нет экспорта данных | Нельзя выгрузить историю | CSV / PDF кнопка |
| Нет настроек уведомлений | Нельзя отключить лишние | Чекбоксы по типам |
| Нет удаления аккаунта | Нарушение 152-ФЗ | Кнопка с confirm dialog |
| Storage: красный/жёлтый | "Fear colors" для расходов | Только primary color |

### Скачивание

| Проблема | Влияние | Решение |
|----------|---------|---------|
| Нет batch download | Скачивать по одному — мучение | ZIP-архив выбранных |
| Нет имени файла | "download" вместо нормального имени | Сохранять оригинальное имя |
| Нет прогресса для больших файлов | Кажется, что зависло | Progress bar |

## Responsive Design

| Viewport | Статус | Проблемы |
|----------|--------|----------|
| Desktop (1440px+) | Отлично | — |
| Laptop (1024px) | Хорошо | Grid работает, ConfigPanel может быть тесно |
| Tablet (768px) | Терпимо | Navbar logo скрыт, некоторые модалки overflow |
| Mobile (375px) | Плохо | DetailPanel не скроллится, Filmstrip обрезан, sidebar не сворачивается, touch targets мелкие |

**Рекомендация:** До запуска не нужен полноценный мобильный UI. Но нужен **минимум:**
- Collapsible sidebar
- Modals → bottom sheets на мобильных
- Touch-friendly кнопки (min 48px)
- Filmstrip → swipe navigation

## Accessibility

| Критерий | Статус | Что нужно |
|----------|--------|-----------|
| Keyboard navigation | Частично | Tab order работает, но не все actions доступны с клавиатуры |
| ARIA labels | Частично | Некоторые иконки без aria-label |
| Color contrast | Хорошо | oklch токены WCAG AA compliant |
| Focus indicators | Хорошо | focus-visible ring есть |
| Screen readers | Плохо | Нет role="status" на loading states, нет live regions |
| Reduced motion | Нет | prefers-reduced-motion не поддержан |
| Form validation | Слабо | aria-invalid не используется консистентно |

**Рекомендация:** WCAG AA — не блокер для запуска, но нужно для корпоративных клиентов.

## Что НЕ трогать

Эти блоки работают хорошо. Рефакторить "для красоты" — потеря времени:

1. **Element Grid** — DnD, multi-select, bulk actions работают отлично
2. **PromptBar** — auto-resize, image inputs, mode selector — зрелый компонент
3. **WebSocket real-time** — reconnect, event handling — стабильно
4. **Design system** — oklch, dark/light, tokens — консистентно
5. **Auth flow** — JWT, refresh, email verification — production-ready

## Заключение

UX Раскадровки — **на 70% от production-ready.** Ядро (генерация, grid, lightbox, шеринг) работает и выглядит профессионально. Но оставшиеся 30% — onboarding, error handling, кабинет, мобильная адаптация — это разница между "крутой проект" и "продукт, за который платят".

**Минимальный набор для запуска:**
1. Onboarding (бесплатные кадры + empty states + prompt suggestions)
2. Error handling cleanup
3. Стоимость рядом с кнопкой генерации
4. Удаление аккаунта

**Всё остальное можно итерировать после запуска** — при условии, что есть мониторинг и фидбек от первых пользователей.
