# docs/guides/ — Инструкции для пользователей

Здесь лежат руководства, написанные **для людей**, а не для агентов.

## Требования к содержимому

- Пишем **простым понятным языком**. Читатель — не разработчик.
- Технические термины — только когда без них нельзя обойтись, и тогда с пояснением.
- Формат: пошаговые действия. Что нажать, что ввести, что должно произойти.
- Скриншоты приветствуются.
- Каждый файл отвечает на вопрос: «Как мне сделать X?»
