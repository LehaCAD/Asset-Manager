# Сессия 09 — Микро-UX

**Цель:** мелкие, но заметные правки. В сумме дают ощущение полировки.

## BF-09-01 — «Отправили письмо со ссылкой для подтверждения» занимает слишком много места на десктопе

**Симптом:**
> «Слишком огромная эта подсказка. Должно быть умещено в одну строку и не занимать так много места. Это должна быть небольшая штучка, а она прям огромная».

**Где:** страница `/check-email` и/или баннер на дашборде.

**Фикс:**
- Однострочный баннер, компактный шрифт (13-14px).
- Иконка + текст + кнопка «Отправить ещё раз».
- Возможно — закрывашка (×).
- Убрать hero-раскладку, сделать thin banner сверху.

**Дизайн:** согласовать через `pencil-new.pen` перед правкой кода.

## BF-09-02 — Кнопка отправки в PromptBar (фиолетовая) выбивается

**Симптом:**
> «Цвет кнопки отправки сообщения из PromptBar — стиль должен быть другой, примерно такой же как у кнопки выбора картинки. Фиолетовый прям очень контрастирует, как будто не из этого мира».

**Фикс:**
- Привести к стилю icon-button (transparent/muted background, subtle border).
- На hover — акцентная подсветка.
- Акцент primary — только в disabled-состоянии → active (как подсказка «можно отправлять»). Обсудить с пользователем.

**Смежно:** пройтись по всем CTA в PromptBar — чтобы они были когерентны.

## BF-09-03 — Аккордеон: мало отступа до сетки элементов

**Симптом:**
> «Надо чуть-чуть больше расстояния от названия аккордеона и чекбокса аккордеона до сетки элементов. В группах нормально, для элементов маловато».

**Фикс:** увеличить `margin-top` сетки элементов под аккордеоном. Ровно то значение, что в группах, или чуть больше.

## BF-09-05 — Общий чеклист «щепетильности»

После основной сессии — пройтись по:
- [ ] все иконки в шапке — иконки lucide, без разнобоя стилей
- [ ] все кнопки icon-only имеют tooltip
- [ ] все кнопки имеют aria-label
- [ ] все disabled-состояния выглядят одинаково
- [ ] фокус-ring на всех интерактивных элементах

Найденные несоответствия — в `docs/BACKLOG_IDEAS.md`, если не критично для ближайшего релиза.
