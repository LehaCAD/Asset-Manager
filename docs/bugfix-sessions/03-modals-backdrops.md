# Сессия 03 — Модалки и бэкдропы

**Цель:** все модалки корректно затемняют фон (в том числе ConfigPanel), UX создания группы — с преднаполнением имени.

---

## BF-03-01 — ConfigPanel не затемняется при открытии модалок

**Симптом:**
> «Нажимаю „создать группу" на светлой теме — модалка затемняет всё, кроме ConfigPanel. Она остаётся такой какой была. В тёмной теме то же самое».

**Также:**
> «При муте бэкграунда при вызове разных модалок ConfigPanel не мутится».

**Корень (гипотеза):**
- ConfigPanel рендерится вне основного layer stack (portal, fixed position, или `z-index` выше overlay).
- Или overlay модалки имеет ограниченный container.

**Что делать:**
1. Найти `ConfigPanel` в `frontend/components/generation/ConfigPanel.*`.
2. Посмотреть z-index и parent.
3. Решить архитектурно: либо модалки рендерятся в `document.body` (React portal) с overlay поверх всего, либо ConfigPanel понижается по z-index, либо сам блокируется на время открытой модалки.

**Тест-план:** все модалки проекта — create group, create project, share, settings, image input, model selector, bulk action dialogs. ConfigPanel должен быть затемнён во всех.

**Проактивно проверить:** что другие «прилипшие» панели тоже затемняются (PromptBar, Breadcrumbs, шапка workspace).

## BF-03-02 — Преднаполнение имени новой группы

**Симптом:**
> «Когда название группы выбираешь — чтобы там преднаписано уже было „Новая группа". Чтобы можно было нажать просто „добавить" сразу. И кнопочку плюс из кнопки „добавить" при создании группы тоже убрать».

**Действия:**
1. В CreateGroupModal — `defaultValue="Новая группа"` + селект текста при фокусе (`onFocus → e.target.select()`), чтобы можно было либо Enter, либо сразу перепечатать.
2. Кнопка submit внутри модалки — убрать иконку `+`, оставить только текст «Добавить».

**Смежно:** проверить CreateProjectModal — там же применить похожий паттерн (default name «Новый проект»)? Согласовать с пользователем.

## BF-03-03 — Модалка шеринга не закрывается после «создать и скопировать»

**Симптом:**
> «Нажимаешь „создать и скопировать", на мобилке ничего не произошло. Нажимаешь ещё раз — тост „не удалось создать ссылку". Потом заходишь — на самом деле создалась. Непредсказуемое поведение. После нажатия модалка должна закрыться».

**Корень:** подробнее в [Сессии 08](08-sharing-ux.md) — тут только UX-часть модалки:
- После успеха (или копирования) — `onClose()`.
- Дизейблить кнопку на время запроса (избежать двойного клика).
- Показать состояние загрузки внутри кнопки.
