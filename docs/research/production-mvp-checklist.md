## 3. Детали по P0 и P1

### P0-1: Закрыть CORS (S)
**Что:** В `settings.py` заменить `CORS_ALLOW_ALL_ORIGINS = True` на `CORS_ALLOWED_ORIGINS` с конкретными доменами.
```python
CORS_ALLOW_ALL_ORIGINS = False
CORS_ALLOWED_ORIGINS = [
    "https://raskadrawka.ru",
    "https://www.raskadrawka.ru",
]
```
**Зависимости:** Нет.

### P0-2: DEBUG и ALLOWED_HOSTS (S)
**Что:** `DEBUG = os.getenv('DEBUG', 'False') == 'True'` (дефолт False). Убрать `0.0.0.0` и dev-хосты из ALLOWED_HOSTS в prod.
**Зависимости:** Нет.

### P0-3: Включить HTTPS (S)
**Что:** Раскомментировать HTTPS секцию в `nginx/default.conf`, запустить `certbot certonly` для `raskadrawka.ru`, включить HTTP→HTTPS редирект.
**Зависимости:** Нет.

### P0-4: Security headers (S)
**Что:** В `settings.py`:
```python
SECURE_SSL_REDIRECT = True
SECURE_HSTS_SECONDS = 31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS = True
SESSION_COOKIE_SECURE = True
SESSION_COOKIE_HTTPONLY = True
SESSION_COOKIE_SAMESITE = 'Lax'
CSRF_COOKIE_SECURE = True
CSRF_COOKIE_HTTPONLY = True
```
В Nginx добавить `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`.
**Зависимости:** HTTPS (#3) должен быть включён.

### P0-5: Rate limiting на auth (M)
**Что:** Добавить DRF throttling в `REST_FRAMEWORK`:
```python
'DEFAULT_THROTTLE_CLASSES': ['rest_framework.throttling.AnonRateThrottle'],
'DEFAULT_THROTTLE_RATES': {'anon': '20/minute'}
```
Плюс per-view throttle на login/register: 5/min.
**Зависимости:** Нет.

### P0-6: Email verification (L)
**Что:**
- Backend: `is_email_verified` поле на User, эндпоинт `/api/auth/verify-email/`, отправка письма при регистрации
- Frontend: страница "подтвердите email", обработка токена из ссылки
- Middleware: блокировать генерацию для неверифицированных
**Зависимости:** Email backend (#8).

### P0-7: Password reset (M)
**Что:**
- Backend: `/api/auth/reset-password/` (отправка ссылки), `/api/auth/reset-password/confirm/` (смена пароля)
- Frontend: страница "забыли пароль", страница ввода нового пароля
**Зависимости:** Email backend (#8).

### P0-8: Email backend (S)
**Что:** Настроить SMTP в `settings.py`. Варианты:
- **Yandex 360 Business** (бесплатно до 1000 писем/сутки, домен `@raskadrawka.ru`)
- **Mailgun** / **SendGrid** (бесплатный tier: 100-200 писем/день)
```python
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.yandex.ru'
EMAIL_PORT = 465
EMAIL_USE_SSL = True
```
**Зависимости:** Нет.

### P0-9: React Error Boundary (S)
**Что:** Создать `app/error.tsx` и `app/global-error.tsx` — стандартные Next.js error boundaries. При краше показывать "Что-то пошло не так" с кнопкой "Обновить".
**Зависимости:** Нет.

### P0-10/11: ToS и Privacy Policy (S)
**Что:** Создать страницы `/terms` и `/privacy` с текстом. Текст можно сгенерировать по шаблону для SaaS-платформы (AI-генерация контента, хранение данных в РФ).
**Зависимости:** Текст документов (можно шаблонный на старт).

### P0-12: Галочка ToS на регистрации (S)
**Что:** Checkbox "Я согласен с [Условиями использования] и [Политикой конфиденциальности]" на форме регистрации. Кнопка disabled без галочки.
**Зависимости:** #10, #11.

### P0-13: Автоматические бэкапы (M)
**Что:**
- Cron-задача: `pg_dump` ежедневно в 3:00
- Хранить 7 последних daily + 4 weekly

- Скрипт `scripts/backup-db.sh`
**Зависимости:** Нет.

### P0-14: Sentry (S)
**Что:**
- `pip install sentry-sdk` в requirements.txt

## 4. Рекомендация по биллинг-провайдеру

### Сравнение

| | ЮKassa | CloudPayments | Tinkoff | Robokassa |
|--|--------|---------------|---------|-----------|
| Комиссия (карты) | 3.5% | 2.3–2.7% | 2.49% | 3.9–5% |
| Комиссия (СБП) | 0.7–3.5% | ~0.7% | от 0.4% | ~1% |
| Ежемесячная плата | 0 | 0 | 0 | 0 |
| Python SDK | **Официальный** | Community | Нет | Нет |
| Рекуррентные платежи | Отлично | Отлично | Хорошо | Базово |
| Встроенная онлайн-касса | Партнёры (АТОЛ) | **CloudKassir (своя)** | Внешняя | Внешняя |
| Apple/Google Pay | SberPay/TPay | **Да** | Tinkoff Pay | Нет |
| Качество API | 9/10 | 7/10 | 7/10 | 5/10 |
| Документация | **Лучшая** | Хорошая | Хорошая | Устаревшая |

### Рекомендация: ЮKassa или CloudPayments

**Вариант А — ЮKassa** (если важна надёжность и скорость интеграции):
- Официальный Python SDK (`pip install yookassa`) — интеграция за 2–3 дня
- Лучшая документация среди РФ-провайдеров
- ~40% рынка РФ — проверенная инфраструктура
- Минус: комиссия чуть выше (3.5%)

**Вариант Б — CloudPayments** (если важна экономия и всё-в-одном):
- Комиссия ниже (2.3–2.7%)
- **CloudKassir** — касса в том же dashboard, одна интеграция
- Apple Pay / Google Pay из коробки
- Минус: нет официального Python SDK

**Моя рекомендация: ЮKassa** для MVP.
- Официальный SDK минимизирует риски и время разработки
- Интеграция с АТОЛ Онлайн хорошо документирована
- Можно переключиться на CloudPayments позже при масштабировании (ради комиссии)

### Юридические требования

| **Онлайн-касса** | Обязательна (54-ФЗ). Облачная касса: АТОЛ Онлайн (~2000–3000 ₽/мес) или CloudKassir (~1500 ₽/мес) |

## 6. Виртуальная валюта "Кадр" — юридика, UX, аналитика

### 6.1 Юридический статус

**"Кадр" — это не виртуальная валюта и не денежный суррогат.** Юридически это **предварительная оплата за услуги** (ГК РФ ст. 487, 781). Кадр — условная единица измерения объёма услуг, а не средство обмена.

**Не подпадает под:**
- 259-ФЗ "О цифровых финансовых активах" — это про крипто, не про внутренние кредиты
- ФЗ-86 ст. 27 "Денежные суррогаты" — Кадры не передаются между пользователями, не выводятся, не используются вне платформы

**Аналоги в РФ:** VK "Голоса" (15+ лет), Steam кошелёк, все gaming-платформы. Работают без проблем.

**Условия безопасности (обязательно соблюдать):**
- Кадры НЕ передаются между пользователями
- Кадры НЕ выводятся как рубли
- Кадры НЕ используются вне raskadrawka.ru
- Кадры НЕ являются средством обмена

### 6.2 Нужна ли вообще виртуальная валюта?

**Да, для данного продукта — однозначно нужна.**

| Причина | Объяснение |
|---------|------------|
| Изоляция от провайдера | Kie.ai поднял цены → меняем стоимость генерации в Кадрах, не трогая рублёвую цену пакетов |
| Кросс-модельный прайсинг | "Фото = 2 Кадра, Видео = 10 Кадров" — понятнее чем разные рублёвые цены |
| Промо-гибкость | "10 бесплатных Кадров при регистрации" проще чем "скидка 50₽" |
| Психология (token effect) | Люди тратят виртуальную валюту свободнее — научно доказано |
| Чеки (54-ФЗ) | Один чек при покупке пакета, а не чек на каждую генерацию |
| Система уже построена | CreditsTransaction, CreditsService, select_for_update — переделка на рубли = бессмысленный рефактор |

**Индустрийный стандарт:** Leonardo.ai (Tokens), Runway (Credits), Midjourney (Fast Hours). Все AI-платформы используют виртуальные единицы.

### 6.3 Аналитика: рубли в личном кабинете

**Показывать рублёвый эквивалент в аналитике — законно.** Дашборд — это не финансовый документ, а статистика использования.

**Рекомендация: показывать ОБА, Кадры — primary, рубли — secondary:**

```
Баланс: 150 Кадров (≈750₽)

Статистика за март:
  Использовано: 87 Кадров (≈435₽)
  Изображений: 62
  Видео: 5
```

**Почему Кадры primary:** в UI генерации цены в Кадрах → консистентность.
**Почему рубли нужны:** прозрачность → доверие. Пользователь понимает реальные траты для бюджетирования.

> **Важно:** Рублёвый эквивалент приблизительный (пользователь мог покупать пакеты по разным ценам). Всегда писать **"≈435₽"** (с "≈"), не "435₽".

**Как делают другие:**
- VK: только Голоса, без рублей
- Steam (РФ): только рубли (кошелёк в рублях)
- SberCloud: рубли для B2B, токены для API

Наш подход прозрачнее, чем у gaming-платформ — это правильно для prosumer-инструмента.

### 6.4 Управление конвертацией

**Фиксированный курс: 1 Кадр = X₽.** Если провайдер меняет цены — меняем стоимость генерации в Кадрах, НЕ курс.

| Действие | Законно? | Примечание |
|----------|----------|------------|
| Изменить цену пакетов для будущих покупок | ✅ Да | Стандартная практика |
| Изменить стоимость генерации в Кадрах | ✅ Да | Нужен пункт в ToS + уведомление |
| Отнять уже купленные Кадры | ❌ Нет | Нарушение ГК РФ ст. 310 |
| Изменить курс задним числом | ❌ Нет | Нарушение прав потребителей |

**В ToS обязательно:**
> "Стоимость пакетов Кадров устанавливается на момент покупки и не подлежит изменению после оплаты. Стоимость услуг генерации в Кадрах может изменяться. Актуальные тарифы публикуются на сайте."

### 6.5 Возвраты

По ст. 26.1 Закона о ЗПП (дистанционные продажи) пользователь имеет право на возврат.

**Рекомендуемая политика:**
- Неиспользованные Кадры возвращаются по курсу покупки **в течение 14 дней**
- Бонусные/подарочные Кадры не возвращаются
- Возврат тем же способом оплаты в течение 10 рабочих дней

**Опционально:** срок действия Кадров (12 месяцев неактивности). Юридически допустимо при указании в ToS. Финансово важно — неиспользованные кредиты = обязательство на балансе.

### 6.6 Чек (54-ФЗ)

Чек выбивается при **покупке пакета** (не при каждой генерации):

```
Предмет расчёта: "Пакет услуг генерации контента с использованием ИИ (100 Кадров)"
Признак предмета расчёта: Услуга
НДС: без НДС (при УСН)
```

Это стандартный подход — так делают VK, Steam, все gaming-платформы.
