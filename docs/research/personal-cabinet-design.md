# Личный кабинет — Проектирование

> Результат ресёрча и проектирования личного кабинета для Раскадровки.

## 1. Ресёрч конкурентов

### Сводная таблица

| Секция | Leonardo.ai | Runway | Midjourney | Figma | Notion | Vercel |
|--------|-------------|--------|------------|-------|--------|--------|
| **Профиль** | Имя, NSFW-тоггл | Имя воркспейса, иконка | Только план и email | Имя, email, пароль, 2FA, аватар | Имя, email, аватар, пароль | Имя, email, аватар |
| **Биллинг** | Подписка + PAYG top-up + авто-пополнение | План, покупка кредитов, инвойсы | План, дата продления, отмена | План, места, инвойсы | План, места, инвойсы, метод оплаты | План, инвойсы, лимиты трат |
| **Кредиты/Использование** | Счётчик токенов на главной (hover = детали), rollover | Баланс в углу + графики по инструментам | Остаток Fast-времени, GPU-часы | AI-кредиты в биллинге | Раздел Usage | Bandwidth, build minutes — визуальные метры |
| **Аналитика** | Нет | **Графики трат по времени и по инструментам** | Нет | Нет (только Enterprise) | Только Enterprise | Observability (логи, аналитика скорости) |
| **Хранилище** | Не показано | Библиотека ассетов, без метра | Не показано | На уровне орг-ции | Не показано | Лимиты bandwidth |
| **API-ключи** | До 10 ключей, авто-пополнение для API | Отдельный домен (docs.dev) | Нет | Personal Access Tokens (scope, expiry) | Integrations / API tokens | Tokens (scoped per team, expiry) |
| **Уведомления** | Нет | Email по биллингу | Нет | Гранулярные email-уведомления | Email, mobile, desktop тогглы | Email + dashboard при лимитах |
| **Безопасность** | Удаление аккаунта | Нет | Нет | 2FA, токены, подключённые приложения | Sharing controls, export, SSO | Attack challenge, deployment protection |

### Ключевые выводы

1. **Баланс всегда на виду** — Leonardo (счётчик внизу), Runway (в углу). Не прятать в настройки.
2. **Аналитика трат — killer feature** — Runway показывает графики по инструментам и по времени. Vercel — визуальные метры по ресурсам. Это то, что хотят пользователи.
3. **Простая структура** — Midjourney: всё на одной странице. Notion: sidebar-в-модалке (масштабируется). Для растущей платформы лучше Notion-подход.
4. **Авто-пополнение** (Leonardo) и **лимиты трат** (Vercel) — premium-фичи для будущего.
5. **Разделение personal / workspace** (Figma, Notion) — планировать заранее, если будут команды.

## 2. Структура секций кабинета

### Навигация (sidebar внутри страницы кабинета)

```
/cabinet
├── /profile          — Профиль
├── /balance          — Баланс и операции
├── /analytics        — Аналитика
├── /storage          — Хранилище
└── /settings         — Настройки
```

### 2.1. Профиль (`/cabinet/profile`)

**Что показываем:**
- Аватар (загрузка / смена)
- Имя пользователя (editable)
- Email (read-only или с подтверждением)
- Смена пароля
- Дата регистрации

**Источник данных:** `User` модель — `username`, `email`, `first_name`, `last_name`, `created_at`

### 2.2. Баланс и операции (`/cabinet/balance`)

**Что показываем:**
- Текущий баланс (крупно, карточка)
- Кнопка "Пополнить" (заглушка до подключения биллинга)
- Pricing percent (если != 100, показать: "Ваша ставка: 80%" → скидка 20%)
- Таблица транзакций с пагинацией

**Таблица транзакций:**

| Колонка | Источник |
|---------|----------|
| Дата | `CreditsTransaction.created_at` |
| Тип | `reason` → человекочитаемое: "Генерация", "Возврат", "Пополнение" |
| Сумма | `amount` (зелёный для +, красный для −) |
| Баланс после | `balance_after` |
| Модель | `metadata.ai_model_name` |
| Элемент | `element` FK → ссылка на элемент |

**Фильтры:**
- Период: сегодня / неделя / месяц / произвольный
- Тип: все / генерация / возврат / пополнение
- Сортировка: по дате (desc по умолчанию)

**Источник данных:** `CreditsTransaction` — `amount`, `balance_after`, `reason`, `metadata`, `element`, `created_at`; `User` — `balance`, `pricing_percent`

### 2.3. Аналитика (`/cabinet/analytics`)

> Самая ценная секция. Детальная проработка — см. раздел 3.

### 2.4. Хранилище (`/cabinet/storage`)

**Что показываем:**
- Progress bar: использовано / лимит (например, "156 МБ из 300 МБ")
- Процент использования
- Топ-5 проектов по объёму (mini bar chart)
- Таблица проектов с размером хранилища

| Колонка | Источник |
|---------|----------|
| Проект | `Project.name` |
| Файлов | `Element.count` per project |
| Объём | `sum(Element.file_size)` per project |
| % от лимита | вычисляемое |

**Источник данных:** `UserQuota.storage_limit_bytes`, `Element.file_size` (агрегация по проектам)

### 2.5. Настройки (`/cabinet/settings`)

**Что показываем:**
- Тема: светлая / тёмная / системная (заглушка, сейчас только тёмная)
- Язык: русский (единственный, но подготовка к i18n)
- Удаление аккаунта (с подтверждением)

### Что НЕ нужно на первом этапе

| Фича | Почему не сейчас |
|------|------------------|
| API-ключи | Нет публичного API |
| Уведомления | Нет системы нотификаций |
| Команды / workspace | Однопользовательская модель |
| Подписки / инвойсы | Биллинг не подключён |
| SSO / 2FA | Избыточно для текущей аудитории |

## 3. Аналитика — детальная проработка

### 3.1. Виджеты верхнего уровня (summary cards)

Четыре карточки в ряд:

| Виджет | Значение | Источник |
|--------|----------|----------|
| **Баланс** | `1,250.00 ⚡` | `User.balance` |
| **Потрачено за период** | `3,420.00 ⚡` | `sum(abs(CreditsTransaction.amount))` where `reason=generation_debit`, filtered by period |
| **Генераций** | `847` | `Element.count` where `source_type=GENERATED`, filtered by period |
| **Хранилище** | `156 / 300 МБ` | `sum(Element.file_size)` / `UserQuota.storage_limit_bytes` |

### 3.2. График: Траты по времени

- **Тип:** Area chart (заполненная линия)
- **Ось X:** дни (при периоде ≤ 31 дня), недели (при периоде ≤ 3 мес), месяцы (при периоде > 3 мес)
- **Ось Y:** сумма трат в Зарядах
- **Данные:** `CreditsTransaction` where `reason=generation_debit`, grouped by `date(created_at)`
- **Overlay:** средняя линия за период (пунктир)
- **Период по умолчанию:** последние 30 дней
- **Селектор периода:** 7 дней / 30 дней / 90 дней / всё время / произвольный

### 3.3. График: Траты по моделям

- **Тип:** Horizontal bar chart или donut chart
- **Данные:** `CreditsTransaction` joined через `element.ai_model`, grouped by `ai_model.name`
- **Метрика:** сумма трат + количество генераций
- **Сортировка:** по сумме трат desc
- **Показываем:** топ-5 моделей + "Остальные"

### 3.4. График: Траты по проектам

- **Тип:** Horizontal bar chart
- **Данные:** `CreditsTransaction` joined через `element.project`, grouped by `project.name`
- **Метрика:** сумма трат + объём файлов
- **Показываем:** топ-5 проектов + "Остальные"

### 3.5. Виджет: Статистика генераций

| Метрика | Источник |
|---------|----------|
| Всего генераций | `Element.count(source_type=GENERATED)` |
| Успешных | `Element.count(status=COMPLETED, source_type=GENERATED)` |
| Неудачных | `Element.count(status=FAILED, source_type=GENERATED)` |
| % успеха | вычисляемое |
| Средняя стоимость | `avg(abs(CreditsTransaction.amount))` where `reason=generation_debit` |
| Самая используемая модель | `mode(Element.ai_model)` |

### 3.6. Фильтры аналитики

| Фильтр | Значения | По умолчанию |
|--------|----------|--------------|
| Период | 7d / 30d / 90d / all / custom | 30d |
| Проект | Все / конкретный | Все |
| Тип контента | Все / Изображения / Видео | Все |
| Модель | Все / конкретная | Все |

Фильтры влияют на все виджеты и графики одновременно.

### 3.7. Группировка данных

- **По времени:** автоматическая гранулярность в зависимости от периода (дни → недели → месяцы)
- **По моделям:** всегда топ-5 + "Остальные" для читаемости
- **По проектам:** всегда топ-5 + "Остальные"
- **По типу:** IMAGE vs VIDEO (для donut-chart или split view)

## 5. Новые API endpoints

### Необходимые endpoints

| Endpoint | Метод | Описание | Источник данных |
|----------|-------|----------|-----------------|
| `/api/cabinet/transactions/` | GET | Список транзакций с пагинацией и фильтрами | `CreditsTransaction` |
| `/api/cabinet/analytics/` | GET | Агрегированная аналитика за период | `CreditsTransaction` + `Element` |
| `/api/cabinet/storage/` | GET | Использование хранилища по проектам | `Element.file_size` + `UserQuota` |
| `/api/auth/me/password/` | POST | Смена пароля | `User` |
| `/api/auth/me/avatar/` | POST | Загрузка аватара | `User` (новое поле) |
| `/api/auth/me/` | DELETE | Удаление аккаунта | `User` |
