# Блок 6: UI/UX дизайн

> Задача из agent-tasks.md: #4 (Графика и UI-компоненты)
> Зависимости: нет — делается параллельно с любым блоком
> Область: только дизайн через MCP Pen. Ничего не интегрировать до согласования.

## Цель

Нарисовать и согласовать визуальные компоненты до их реализации в коде.

---

## Что нарисовать

### 1. Model Select — карточки моделей

Текущее состояние: `ModelCard.tsx` — простые карточки с preview_url, name, description, tags.

Нарисовать:
- Карточка модели с красивым превью (графика, не фото)
- Состояния: default, hover, selected
- Бейджи (tags) — стиль
- Разделение IMAGE / VIDEO моделей визуально

### 2. Выбор первого и последнего кадра

Для VEO-подобных моделей:
- Попап выбора режима: карточки "Начальный кадр", "Конечный кадр", "Референсы"
- Иконографика для каждого режима
- Состояние "выбрано" с миниатюрой выбранного кадра

### 3. Выбор референсов

- Модалка "Выберите до N референсных изображений"
- Grid выбора с чекбоксами/highlight
- Счётчик "3/3 выбрано"
- Пустое состояние

### 4. Кнопки разрешения

Крупные кнопки как в Leonardo AI:
- `16:9` — горизонтальный прямоугольник
- `9:16` — вертикальный прямоугольник
- `1:1` — квадрат
- Визуально — пропорции видны в форме самой кнопки
- Состояния: default, hover, selected

### 5. Статус-оверлей на карточке (Frame.io стиль)

- Карточка с overlay снизу
- Показывает: модель, время, статус
- Варианты: generating, uploading, failed

### 6. Баланс в navbar

- Компактное отображение баланса
- Ненавязчивый стиль
- Обновление анимация (число уменьшается)

### 7. Метрики проекта

- Панель или карточки со статистикой проекта
- Затраты, элементы, память
- Компактный вариант + развёрнутый

---

## Стилистические принципы

- Dark theme (основная тема приложения)
- Цветовая палитра: из текущего `globals.css` (shadcn/tailwind)
- Иконки: lucide-react (уже используется)
- Скругления, тени — как в существующих компонентах
- Шрифт: тот же что в проекте

## Процесс

1. Нарисовать через MCP Pen
2. Показать владельцу на согласование
3. Итерации правок
4. Финальное "ок" → можно интегрировать

## Чего НЕ делать

- Не писать код
- Не менять существующие компоненты
- Не интегрировать в проект до согласования
