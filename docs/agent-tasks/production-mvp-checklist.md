## Контекст

**Раскадровка** (`raskadrawka.ru`) — AI-продакшен платформа. Уже работает на VPS с живыми данными, но пользователи заходят по приглашению, кредиты начисляются вручную через Django Admin.

Цель: определить минимальный набор фич и доработок, чтобы можно было:
1. Прикрутить биллинг (оплата → кредиты автоматически)
2. Открыть регистрацию для всех
3. Дать людям тестировать без условностей типа "переведи на карту, а я токенов накину через админку"

**Важно:** Сам биллинг (интеграция с платёжкой) пока НЕ делаем. Но всё вокруг него должно быть готово — чтобы его подключил и всё, готово.

## Что уже есть

### Работает:
- JWT авторизация (login, register, refresh)
- CRUD проектов, групп, элементов
- AI-генерация через Kie.ai (polling + webhook)
- Кредитная система ("Заряд"): дебит при генерации, рефунд при ошибке, идемпотентность
- Presigned upload в S3
- WebSocket real-time обновления
- Schema-driven UI для параметров моделей
- Pricing per model (fixed + lookup)
- Публичный шеринг (базовый — SharedLink + Comments, models есть, UI начат)

### Не работает / не готово:
- Нет самостоятельной оплаты (только ручной topup через админку)
- Нет email-подтверждения при регистрации
- Нет восстановления пароля
- Нет личного кабинета (проектируется параллельно, Задача B)
- Шеринг начат, но не доведён до конца
- Нет rate limiting на API
- Нет мониторинга / alerting
- Нет Terms of Service / Privacy Policy
- Нет onboarding flow для новых пользователей

## Что нужно сделать

### 1. Gap-анализ
Проведи ресёрч: что нужно SaaS-платформе для запуска в продакшен. Раздели на категории:

**Безопасность:**
- Что обязательно (auth, rate limiting, CORS, CSRF, input validation...)
- Что есть, что нет

**Юридика:**
- Terms of Service, Privacy Policy, Cookie consent
- Что нужно для работы с платежами в РФ
- GDPR / ФЗ-152 (персональные данные)

**Инфраструктура:**
- Мониторинг (что мониторить, какие инструменты)
- Бэкапы (текущее состояние, что улучшить)
- Логирование (structured logging, ротация)
- Масштабирование (bottlenecks при росте)

**Пользовательский опыт:**
- Onboarding (что видит новый пользователь)
- Email-система (подтверждение, восстановление пароля, уведомления)
- Error handling (что видит пользователь при ошибках)

**Биллинг-ready:**
- Что должно быть готово ДО подключения платёжки
- Какие платёжные системы подходят для РФ (ЮKassa, CloudPayments, Tinkoff, Stripe)
- Какие webhook/API нужны
- Как связать "оплату" → "начисление Зарядов"
- Нужны ли тарифные планы или только pay-as-you-go?

### 2. Приоритизация
Раздели всё на:
- **P0 (блокер запуска)** — без этого нельзя открывать
- **P1 (нужно в первый месяц)** — можно запуститься без, но нужно быстро добавить
- **P2 (хорошо бы)** — улучшения, но не критичны

### 3. Конкретный чеклист
Для каждого P0 и P1 пункта:
- Что конкретно сделать
- Примерная сложность (S/M/L)
- Зависимости (от чего зависит, что блокирует)

### 4. Рекомендация по биллинг-провайдеру
На основе ресёрча рекомендуй конкретного платёжного провайдера для РФ-рынка с обоснованием:
- Стоимость подключения / комиссия
- Поддержка рекуррентных платежей
- SDK / API качество
- Подходит ли для модели "купить пакет кредитов"
