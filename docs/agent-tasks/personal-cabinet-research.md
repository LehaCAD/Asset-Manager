# Задача B: Проектирование личного кабинета

> **ПРАВИЛО:** В начале каждого ответа кратко напоминай, чем занимаешься.
> Формат: `[Задача B: Личный кабинет] — <что делаю сейчас>`
> Это нужно потому что параллельно работают 4 агента и пользователь должен понимать кто есть кто.

## Контекст

**Раскадровка** — AI-продакшен платформа. Сейчас личного кабинета нет вообще. Есть только:
- `User` модель с полями: email, balance (кредиты "Заряды"), pricing_percent
- `UserQuota` модель: max_projects, max_scenes, max_elements, storage_limit
- Эндпоинт `GET /api/auth/me/` — возвращает профиль
- Эндпоинт `PATCH /api/auth/me/` — обновление профиля

Биллинг пока не подключен, но будет. Валюта — "Заряд" (не рубли, по юридическим причинам).

Данные которые уже есть в системе:
- `CreditsTransaction` — журнал всех операций (дебит, рефунд, топап) с привязкой к элементам
- `Element` — каждый сгенерированный/загруженный файл, привязан к проекту и группе, имеет file_size
- `Project` — проекты пользователя с датами
- `AIModel` — какие модели использовались (через element.ai_model FK)

## Что нужно сделать

### 1. Ресёрч лучших практик
Исследуй как устроены личные кабинеты у SaaS-платформ (особенно AI-генерация, креативные инструменты):
- Leonardo.ai
- Runway
- Midjourney
- Figma
- Notion
- Vercel

Для каждого зафиксируй: какие секции, какие данные показывают, что удобно, что нет.

### 2. Определение секций кабинета
На основе ресёрча и имеющихся данных предложи структуру личного кабинета. Вот что точно нужно:

**Обязательно:**
- Профиль (аватар, имя, email, смена пароля)
- Баланс и история операций (транзакции, фильтры по дате/типу)
- Квоты и лимиты (сколько использовано / сколько доступно)
- Хранилище (сколько S3 занято, прогресс-бар)

**Из идей заказчика:**
- Аналитика по проекту: сколько потрачено, объём файлов, распределение трат по дням, какие модели использовались
- Общая аналитика аккаунта

**На твоё усмотрение (ресёрч):**
- Что ещё может понадобиться? API ключи? Уведомления? Тема? Язык?
- Что точно НЕ нужно на первом этапе?

### 3. Аналитика — детальная проработка
Это самая интересная часть. Из имеющихся данных можно построить:
- Траты по дням/неделям/месяцам (из CreditsTransaction)
- Траты по моделям (через element.ai_model)
- Траты по проектам (через element.project)
- Объём хранилища по проектам (через element.file_size)
- Количество генераций по статусам (completed/failed)
- Средняя стоимость генерации
- Самая используемая модель

Предложи:
- Какие графики/виджеты показывать
- Какой период по умолчанию
- Как группировать данные
- Какие фильтры нужны

### 4. Wireframe-описание
Для каждой секции опиши текстово:
- Какие элементы UI (таблицы, графики, карточки, прогресс-бары)
- Какие данные откуда берутся (укажи конкретные модели/поля)
- Какие API endpoint понадобятся
