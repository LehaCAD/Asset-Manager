# Блок 2 — Визуальный редизайн кабинета

> Дизайн-сессия: mockup в pen → согласование → код. Может затронуть брендбук.

## Контекст

Личный кабинет выглядит невзрачно, плоско, неотличимо. На обеих цветовых схемах — скудно. Не тянет на премиум. Выглядит как работа джуна.

## Задачи

### 1. Формы и цвета кабинета

Нужно поработать с формой и цветами всех разделов кабинета. Предложить решения визуально — рисовать mockup в pen-файле, согласовывать с пользователем, итерировать пока не будет нужный результат.

**Разделы кабинета:** аналитика, журнал генераций, баланс, хранилище, уведомления, настройки.

### 2. Общий аудит цветов на обеих темах

Пройтись по всем цветам кабинета на тёмной и светлой теме. Привести к премиум-уровню. Если по результатам нужно скорректировать брендбук — сделать это.

## Подход

1. Открыть `pen/pencil-new.pen`, изучить текущую дизайн-систему (Color System v2 — node `gx1cz`)
2. Нарисовать mockup обновлённого кабинета в pen
3. Показать пользователю, собрать обратную связь
4. Итерировать до утверждения
5. Перенести в код
