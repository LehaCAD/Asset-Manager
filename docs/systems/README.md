# docs/systems/ — Документация подсистем

Здесь описано **актуальное состояние** каждой подсистемы сервиса.

## Правила

- Это **не лог изменений**. Здесь отражается текущее устройство, а не история.
- Каждый файл описывает одну подсистему: модели, сервисы, API, админку, фронтенд.
- При внесении правок в подсистему — обновляй соответствующий файл.
- Новый агент должен прочитать файл перед работой с подсистемой.

> **Feature gating — обязательная увязка с подписками.**
>
> Каждая фича в продукте гейтится через систему подписок (`docs/systems/subscriptions.md`).
> При добавлении новой фичи агент **обязан**:
>
> 1. Создать `FeatureFlag` в бэкенде (модель, миграция, привязка к тарифам).
> 2. Добавить проверку `useFeatureGate("<code>")` на фронтенде — заблокированное состояние с `TierBadge` и `UpgradeModal`.
> 3. Добавить backend enforcement в соответствующем view/сервисе (проверка `has_feature`).
> 4. Обновить таблицу гейтов в `docs/systems/subscriptions.md` (секция «Точки интеграции»).
>
> Фича без гейтинга — это баг. Не бывает фич, доступных всем тарифам по умолчанию, если это явно не решено.
> Подробности механики: [subscriptions.md](subscriptions.md) — секции «Feature Gating», «Хук useFeatureGate», «Точки интеграции».
