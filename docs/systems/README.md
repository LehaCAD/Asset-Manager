# docs/systems/ — Документация подсистем

Здесь описано **актуальное состояние** каждой подсистемы сервиса.

## Правила

- Это **не лог изменений**. Здесь отражается текущее устройство, а не история.
- Каждый файл описывает одну подсистему: модели, сервисы, API, админку, фронтенд.
- При внесении правок в подсистему — обновляй соответствующий файл.
- Новый агент должен прочитать файл перед работой с подсистемой.

## Содержание

|------|------------|

| [topup-payments.md](topup-payments.md) | Пополнение баланса через ЮKassa |
| [sharing.md](sharing.md) | Шеринг, комментарии, реакции, согласования, отзывы |
